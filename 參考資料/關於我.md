## Design system — DO NOT DRIFT

The look is a dark **"instrument / status panel."** It deliberately avoids the generic dark-portfolio cliché (pure black + acid green). Keep that intent.

**Signature element:** the live status panel in the hero (`.panel`) — boot-reveal rows, live GMT+8 clock, LIVE pill. This is the *one* bold move. Everything else stays quiet. Spend boldness only here.

## Content principles — the important part

This was the hard-won lesson of the project. Preserve it.

1. **Evidence over vision.** Specificity = credibility (especially for an 18-year-old). Every claim should be concrete and verifiable. Cut abstract filler ("reduce friction", "decision cost", "information flow") unless it's immediately backed by something real.
2. **Lead with the security origin story.** Found real vulnerabilities → responsibly disclosed → brought into the computer center to fix them → now does authorized security work. Strongest signal on the site (capability + ethics + initiative). Don't bury it.
3. **Real numbers + links.** Agora-AI: reverse-engineered an undocumented school API; 10 agent tools; 69 passing tests; FastAPI / Next.js / SQLite / Docker; deploying via Cloudflare Tunnel. Keep metrics real and current (the `.metric` chips).
4. **Mindoyo is NOT a company.** It's the honest umbrella for actually-shipped projects. Never inflate it into a startup — the owner explicitly rejects fake-company packaging.
5. **Keep a little personality.** Owner is INTJ, a bit wild, dislikes overly serious settings. One or two wry, specific lines per page (e.g. "a bug I probably shouldn't have found", "since the disclosure"). Don't turn it corporate-sober.
6. **AI Fitness Engine is demoted** to a side experiment — not Featured work. Don't promote it back.

## Safety guardrail (non-negotiable)

This is a **public** page about security work on a **named institution's** systems.

- **Never** publish exploitable detail: no specific endpoints, payloads, current live vulnerabilities, or step-by-step.
- **OK:** vulnerability *classes* as competency proof (XSS, IDOR, clickjacking, credential exposure), always framed as found-and-remediated / authorized / responsible.
- When in doubt, say less.

## Quality floor (keep it passing)

- Responsive down to ~360px (owner browses on mobile). Check the panel rows + `.stats` strip stacking.
- `prefers-reduced-motion` respected — boot reveal, scroll reveals, pulse all gated. Keep it.
- Visible keyboard focus via `:focus-visible`. Keep it.
- Don't break the live clock (GMT+8) or the IntersectionObserver reveal + fallback.
- No `localStorage` / `sessionStorage` if ever rendered as a Claude artifact. (In the owner's own Next.js host, browser storage is fine.)

# Designing systems that disappear

好的工具不會讓你感受到它的存在——它只是讓你把事情做完。

這是我在設計 Agora-AI 時想得最多的一件事。學校的 API 沒有文件、介面散落在七個不同的系統裡——學生每天都在解決「怎麼找到資訊」的問題，而不是「我想做什麼」的問題。

## 問題不在功能，在摩擦

當一個系統需要你記住它的邏輯，它就失敗了。

Agora-AI 的核心想法很簡單：學生說想做什麼，系統去找方法。不是「請選擇功能 → 輸入學號 → 確認」，而是「我下週有哪些課？」然後得到答案。

## 消失的代價

讓介面消失是很貴的。你要做更多的工作——逆向工程、容錯、語義理解——才能讓使用者感受到「這很簡單」。

但這是對的代價。

用戶的心智模型裡不該有你的系統架構。他們只是想完成事情。
